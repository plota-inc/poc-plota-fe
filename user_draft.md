poc 구현 목표:
1. A4 최대 20장 이하의 글쓰기를 한다
2. 드래그 앤 드랍으로 선택한 영역의 수정을 위해 글쓰기 영역 우측에서 ai의 초안을 받아본다 (https://sudowrite.com/ 참고)
3. 초안을 바로 글쓰기 영역으로 집어넣는다.

기술 주요 목표:
Llama-3.2-3B 를 브라우저 상에서 다운받아 IndexedDB에 저장
IndexedDB에 로컬작업 파일과 기능을 저장

특이사항: 
오프라인 저장 모드
pwa 앱으로 배포

기술스택:
1. 코어 프레임워크: Next.js 16+ (App Router)
이유: 서버 사이드 렌더링(SSR)을 통해 첫 화면 로딩을 줄이고, Server Actions를 통해 NestJS 백엔드와 가장 안전하게 통신합니다.
핵심: AI 연산은 브라우저에서 하되, 유저 데이터와 보안 로직은 서버에서 관리하는 하이브리드 구조에 최적입니다.
2. 에디터 엔진: TipTap (ProseMirror 기반)
이유: Sudowrite가 선택한 ProseMirror의 강력함을 그대로 가져가면서, React와의 궁합이 가장 좋습니다.
커스텀: "사전 연결", "AI 말투 제안 하이라이트" 같은 복잡한 기능을 Extension 형태로 모듈화하기 매우 편리합니다.
3. 로컬 AI 엔진: Transformers.js v3+
이유: 2026년 기준 WebGPU 지원이 완벽해졌으며, WASM 대비 최대 100배 빠른 속도를 냅니다.
기능: Llama-3.2-3B, Gemma-2 같은 최신 소형 모델을 브라우저에 캐싱(IndexedDB)하여 서버 비용 없이 무제한으로 사용 가능하게 합니다.
4. 멀티스레딩 통신: Comlink
이유: 메인 화면이 AI 연산 때문에 버벅이는 것을 방지하기 위해 Web Worker를 쓰는데, 이 워커와의 대화를 일반 함수 호출처럼 만들어줍니다.
장점: 개발 난이도가 확 낮아지고 코드가 깔끔해집니다.

확장성:
poc 이후 텍스트 생성 외에 이미지 생성, 나레이션 음성 생성 등 추가 확장될 여지가 있습니다.
